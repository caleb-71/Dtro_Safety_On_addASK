원인 딱 잡혔습니다. ✅
지금 인덱싱이 실패한 이유는 **임베딩(embedding) 모델이 Ollama에 설치(다운로드)되어 있지 않아서**예요.

에러 메시지 그대로 해석하면:

* 인덱싱 과정에서 `http://localhost:11434/api/embeddings` 호출함
* 요청에 사용한 모델이 `nomic-embed-text`
* 그런데 **Ollama에 그 모델이 없음**
* 그래서 404가 뜸 → 인덱싱 실패

---

# ✅ 해결: 임베딩 모델을 Ollama에 “pull(다운로드)” 하면 끝

## 1) PowerShell에서 아래 실행 (프로젝트 루트가 아니어도 됨)

```powershell
ollama pull nomic-embed-text
```

다운로드가 끝나면 확인:

```powershell
ollama list
```

목록에 `nomic-embed-text`가 보이면 성공입니다.

# 🔍 추가로 중요한 설명 (초보자용)

## 왜 LLM(예: llama3.1:8b)만 있으면 안 되나요?

RAG는 모델이 2개 역할로 나뉩니다.

1. **임베딩 모델(검색용)**

   * 문서/질문을 숫자 벡터로 바꾸는 모델
   * 예: `nomic-embed-text`

2. **LLM(답변 생성용)**

   * 검색된 근거를 읽고 최종 답변을 쓰는 모델
   * 예: `llama3.1:8b`

👉 지금은 1번이 없어서 인덱싱이 막혔던 겁니다.

---

# 🛠️ 만약 다른 임베딩 모델을 쓰고 싶다면?

Ollama 임베딩 모델은 여러 개가 있고, 프로젝트 설정(`settings.yaml` 또는 `core/config.py`)에 모델명이 있을 가능성이 큽니다.

하지만 **가장 간단/안정적인 기본값은 `nomic-embed-text`**라서
지금처럼 pull로 해결하는 게 가장 빠릅니다.
