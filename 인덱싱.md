아주 **핵심을 정확히 찌른 질문**입니다.
이 부분을 헷갈리면 RAG 시스템이 “느려지거나, 관리가 어려워지거나, 서버가 터집니다.”

결론부터 명확히 말하고, 그 다음에 **왜 그런지 / 어떻게 설계하는 게 정석인지**를 설명할게요.

---

# 결론 (정답)

> **PDF를 넣자마자 자동으로 매번 인덱싱하지 않는다.
> 반드시 “미리 인덱싱(사전 인덱싱)”을 한다.
> 단, ‘필요할 때만’ 인덱싱을 실행한다.**

즉,

* ❌ 질의할 때마다 자동 인덱싱 → **잘못된 설계**
* ❌ 앱 실행할 때마다 무조건 인덱싱 → **잘못된 설계**
* ✅ **관리자가 명시적으로 인덱싱 실행** → **정석**

---

# 왜 “자동 인덱싱”을 하면 안 되는가?

## 1️⃣ 인덱싱은 “무거운 작업”이다

PDF 인덱싱은 내부적으로 다음을 합니다:

1. PDF 파일 열기
2. 페이지별 텍스트 추출
3. 문단 청킹
4. 임베딩 모델 호출 (수백~수천 번)
5. 벡터 인덱스 저장(FAISS)
6. 메타데이터 기록

👉 이건 **질의(Q&A)**와 다릅니다.
👉 **CPU/메모리/시간을 많이 쓰는 작업**입니다.

그래서:

* 사용자가 질문할 때마다 실행 ❌
* 서버 시작할 때마다 실행 ❌

---

## 2️⃣ 실무 RAG의 표준 패턴은 “사전 인덱싱”

### RAG는 항상 **2단계**입니다

```
[1단계] 인덱싱 (Offline / 관리 작업)
[2단계] 질의     (Online / 사용자 요청)
```

* 인덱싱: 문서가 바뀔 때만
* 질의: 인덱스를 사용해서 즉시 검색

이 패턴은:

* OpenAI 문서
* LangChain
* LlamaIndex
* 기업 RAG 시스템

👉 **전부 동일합니다**

---

# DTRO-Safety-On에서의 “정석 설계”

## ✅ 1단계: PDF를 폴더에 넣는다

```
data/docs/
  laws/
  regulations/
  guidelines/
```

👉 이 시점에서는 **아직 RAG 아님**

---

## ✅ 2단계: “인덱싱 실행”을 한 번 한다

### 방법 1️⃣ (개발/운영 표준)

**관리자용 스크립트 실행**

```bash
python dataops/pipelines/build_index.py
```

### 방법 2️⃣ (UI 기반, 추천)

**Streamlit 관리자 페이지**

```
[규정 인덱스 업데이트] 버튼 클릭
```

이때 내부적으로:

* 새 PDF / 변경된 PDF만 인덱싱
* 기존 인덱스는 재사용

---

## ✅ 3단계: 질의 시에는 “검색만”

```text
사용자 질문
 → 질문 임베딩
 → FAISS 검색 (0.01~0.1초)
 → LLM 답변 생성
```

👉 **인덱싱 없음**

---

# 그럼 “완전 자동”은 언제 쓰는가?

### ❗ 예외적인 경우만 자동 인덱싱

아주 제한적인 상황에서만 자동화합니다.

### 예외 1️⃣ 배치 서버

* 매일 새벽 2시
* 규정 문서가 자동 수집됨
* 인덱싱도 같이 배치로 실행

### 예외 2️⃣ 문서 수집 파이프라인

* DMS(문서관리시스템) 연동
* 새 문서 승인 시 이벤트 발생
* 그 이벤트로 인덱싱 실행

👉 **사용자 질의와는 절대 연결하지 않는다**

---

# “그럼 코드에서는 어떻게 구현하나?”

## 핵심 구조 (중요)

```text
rag/
  indexing/        ← 인덱싱 전용
    build_index.py
    update_index.py
  retrieval/       ← 질의 전용
    retriever.py
```

* 질의 코드에는 **절대 PDF 로딩/임베딩 없음**
* 인덱싱 코드는 **질의 코드와 완전히 분리**

---

# 체크섬(checksum)을 쓰는 이유 (아주 중요)

> “PDF가 수십 개인데, 매번 전부 다시 인덱싱해야 하나요?”

❌ 아닙니다.

### 체크섬 방식

* PDF 파일의 해시값을 저장
* 이전 해시와 다를 때만 인덱싱

```json
{
  "철도안전법_2024.pdf": "a8c93f...",
  "사내안전관리규정_v3.pdf": "9f13d2..."
}
```

이렇게 하면:

* 새 파일 → 인덱싱
* 수정된 파일 → 재인덱싱
* 그대로인 파일 → 스킵

👉 **운영 필수 기능**

---

# DTRO-Safety-On 기준 “정답 시나리오”

### ❓ “이 코드를 돌리면 자동으로 실행되나?”

👉 **아니요. 자동으로 하지 않는 게 정답입니다.**

### ✔ 올바른 흐름

1. PDF를 `data/docs/`에 넣는다
2. 관리자만 인덱싱 실행

   * 스크립트 또는 UI 버튼
3. 인덱스 생성 완료
4. 모든 사용자는 **즉시 질의 가능**
5. 문서 변경 시에만 다시 인덱싱
